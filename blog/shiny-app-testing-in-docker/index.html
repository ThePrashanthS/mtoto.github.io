Before moving on to containers and stuffs, there is another dope tool we need to discuss: The package shinytest, an automated testing agent for, you guessed it‚Ä¶shiny apps. It works as follows:

Shinytest uses snapshot-based testing strategy. The first time it runs a set of tests for an application, it performs some scripted interactions with the app and takes one or more snapshots of the application‚Äôs state. These snapshots are saved to disk so that future runs of the tests can compare their results to them.

The interface could not be easier. You install the package and when the first iteration of your shiny app is ready to roll, you simply run recordTest():

devtools::install_github("rstudio/shinytest")
library(shinytest)

recordTest("path/to/app")

This launches an iframe consisting of your shiny app where you can control what to test. Each interaction with the dashboard is recorded, and when you hit Take snaphsop, the state of your dashboard is recorded:
